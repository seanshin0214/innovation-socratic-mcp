## 1. 도구 개요

### 정의
브레인스토밍은 그룹이 자유롭게 아이디어를 제시하며, 비판 없이 가능한 많은 아이디어를 생성하는 창의적 발상 기법

### 핵심 원리 (4가지 규칙)
1. **비판 금지 (Defer Judgment)**: 평가는 나중에
2. **자유분방 (Wild Ideas)**: 미친 아이디어 환영
3. **양 추구 (Quantity)**: 많을수록 좋다
4. **조합 발전 (Build On)**: 다른 아이디어 위에 쌓기

### 언제 사용하나?
- 새로운 아이디어가 필요할 때
- 문제 해결책 탐색
- 프로젝트 초기 단계
- 팀 창의성 활성화
- 다양한 관점 수집
- 혁신 워크샵

---

## 2. 진행 단계

### Step 1: 준비 (5분)
- 주제/질문 명확히 정의
- 화이트보드/포스트잇 준비
- 퍼실리테이터 지정
- 4가지 규칙 공유

### Step 2: 워밍업 (5분)
- 간단한 창의력 게임
- 긴장 완화
- 에너지 높이기

### Step 3: 아이디어 생성 (15-20분)
- 모든 아이디어 기록
- 침묵도 OK
- 속도보다 흐름
- 아이디어에 번호 부여

### Step 4: 정리 (10분)
- 유사 아이디어 그룹핑
- 중복 제거
- 카테고리 명명

### Step 5: 평가 (별도 세션)
- 기준에 따라 평가
- 투표 또는 매트릭스

## 3. 브레인스토밍 변형 기법

### 3.1 브레인라이팅 (Brainwriting)
침묵 속 글로 작성

| 단계 | 활동 | 시간 |
|------|------|------|
| 1 | 종이에 3개 아이디어 작성 | 3분 |
| 2 | 옆 사람에게 전달 | - |
| 3 | 받은 아이디어에 3개 추가 | 3분 |
| 4 | 5-6회 반복 | 15-18분 |

**장점**: 외향적 인원 독점 방지

### 3.2 라운드 로빈 (Round Robin)
순서대로 한 개씩 제시

```
참가자1 → 참가자2 → 참가자3 → ...
   ↑                           │
   └───────────────────────────┘
```

**장점**: 모두가 동등하게 참여

### 3.3 역 브레인스토밍 (Reverse)
문제를 악화시키는 방법 발상

1. "어떻게 하면 이 문제를 더 나쁘게 만들까?"
2. 아이디어 수집
3. 각 아이디어를 반대로 뒤집기
4. 해결책으로 변환

### 3.4 스타버스팅 (Starbursting)
질문 중심 브레인스토밍

```
        WHO?
          │
  WHY? ── ★ ── WHAT?
          │
       WHERE?
    WHEN?  HOW?
```

아이디어 대신 질문 생성

### 3.5 SCAMPER
체계적 변형 질문

| 문자 | 의미 | 질문 |
|------|------|------|
| S | Substitute | 무엇을 대체? |
| C | Combine | 무엇을 결합? |
| A | Adapt | 어떻게 적용? |
| M | Modify | 어떻게 수정? |
| P | Put to use | 다른 용도? |
| E | Eliminate | 무엇을 제거? |
| R | Reverse | 뒤집으면? |

### 3.6 6-3-5 방법
6명, 3아이디어, 5분

```
라운드 1: 각자 3개 아이디어 (5분)
라운드 2: 옆으로 전달, 3개 추가 (5분)
... (6라운드)
결과: 6 x 3 x 6 = 108개 아이디어
```

## 4. 퍼실리테이션 팁

### 에너지 관리
- 시작할 때 높은 에너지로
- 정체 시 휴식 또는 변형 기법
- 타이머로 긴박감

### 침묵 대처
- 2분 개인 생각 시간
- "만약에..." 질문 던지기
- 다른 관점 제시

### 독점 방지
- 브레인라이팅 전환
- "아직 말 안 한 분?" 촉진
- 라운드 로빈 사용

### 기록
- 모든 아이디어 가시화
- 번호 부여
- 아이디어 소유권 없애기

---

## 5. 평가 기법

### 5.1 도트 투표 (Dot Voting)
- 각자 스티커 3-5개
- 마음에 드는 아이디어에 부착
- 득표 수로 우선순위

### 5.2 2x2 매트릭스
```
영향력    높│ Quick │ Must  │
          │ Wins  │ Do    │
         낮│ Don't │ Nice  │
          │ Bother│ to    │
          └───────┴───────┘
           쉬움     어려움
             실행 난이도
```

### 5.3 NUF 테스트
- **N**ew: 새로운가?
- **U**seful: 유용한가?
- **F**easible: 실현가능한가?

## 6. 흔한 실수와 해결

| 실수 | 해결 |
|------|------|
| 즉각 비판 | 규칙 재강조, 비판자에게 대안 요청 |
| 아이디어 고갈 | 변형 기법 전환, 휴식 |
| 소수 독점 | 브레인라이팅, 라운드로빈 |
| 주제 이탈 | 초점 질문 다시 공유 |
| 평가 섞임 | 발산-수렴 분리 강조 |

## 8. 질문 시퀀스

### Question 1: 문제 정의
"우리가 해결하려는 것은 정확히 무엇인가?"

### Question 2: 이상적 상태
"완벽하게 해결된 상태는 어떤 모습인가?"

### Question 3: 제약 제거
"아무 제약이 없다면 어떻게 하겠는가?"

### Question 4: 다른 관점
"[고객/경쟁자/아이]라면 어떻게 접근할까?"

### Question 5: 조합
"이 아이디어들을 어떻게 조합할 수 있을까?"

## 9. 주의사항

### Do's
- 4가지 규칙 철저히
- 모든 아이디어 기록
- 양에서 질로 진행
- 발산과 수렴 분리

### Don'ts
- 아이디어 생성 중 평가
- "그건 안 돼" 발언
- 침묵 강요
- 너무 긴 세션 (45분 초과)
