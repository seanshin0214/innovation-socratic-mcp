## 1. 도구 개요

### 정의
TRIZ(트리즈)는 수백만 건의 특허 분석을 기반으로 발명/혁신의 패턴을 추출하여 체계화한 창의적 문제해결 방법론으로, 기술적 모순을 해결하는 40가지 발명원리와 다양한 도구를 제공

### 핵심 원리
- **모순 제거**: 문제의 핵심은 모순이며, 모순 해결이 혁신
- **이상성(Ideality)**: 비용/해로움 최소, 유용 기능 최대인 이상적 해결책 추구
- **자원 활용**: 시스템 내/외부의 활용 가능한 자원 탐색
- **패턴 기반**: 과거 발명 패턴을 현재 문제에 적용

### TRIZ의 핵심 전제
> "누군가는 이미 당신의 문제를 해결했다"

### 언제 사용하나?
- 엔지니어링 문제
- 제품 설계
- 혁신 도전 과제
- 트레이드오프 상황
- "불가능한" 문제
- 기술 최적화

## 2. 5단계 질문 시퀀스

### Question 1/5: 모순 정의
개선하려는 파라미터는 무엇인가? (예: 속도, 강도, 신뢰성, 비용)

개선하려고 할 때 악화되는 파라미터는? (예: 속도 높이면 복잡도 증가)

**목적:** 핵심 트레이드오프 식별

### Question 2/5: 개선/악화 파라미터 분류
TRIZ 39가지 엔지니어링 파라미터에 매핑:
- 이동/정지 객체의 무게
- 길이, 면적, 부피
- 속도, 힘, 파워
- 에너지, 온도, 압력
- 신뢰성, 내구성, 생산성
- 복잡도, 자동화, 제조용이성

**목적:** TRIZ 용어로 문제 프레이밍

### Question 3/5: 모순 매트릭스 참조
매트릭스(개선 파라미터 × 악화 파라미터)에서 TRIZ가 제안하는 발명원리는?

**목적:** 검증된 해결 패턴 찾기

### Question 4/5: 원리 적용
제안된 원리를 타협 없이 모순 해결에 어떻게 적용할 수 있는가? 당신의 맥락에서 [원리]는 어떤 모습인가?

**목적:** 혁신적 해결책 생성

### Question 5/5: 평가 및 반복
이 해결책이 트레이드오프를 진정으로 제거(이상적)하는가, 아니면 최적화(수용가능)만 하는가? 어떤 새로운 모순이 발생하는가?

**목적:** 정제 및 테스트

## 5. 물리적 모순 해결

**물리적 모순**: A가 동시에 있어야 하고 없어야 하는 상황

| 전략 | 설명 | 예시 |
|------|------|------|
| 시간 분리 | 다른 시간에 상반된 속성 | 접이식 우산 |
| 공간 분리 | 다른 공간에 상반된 속성 | 연필 (나무+흑연) |
| 조건 분리 | 다른 조건에서 상반된 속성 | 감온 잉크 |
| 전체/부분 분리 | 전체와 부분이 상반 | 체인 (유연+강함) |

## 7. 예시: 커피컵 문제

### 문제
"커피컵을 들고 다닐 때 흔들리면 커피가 쏟아진다"

### 모순 식별
- **기술적 모순**: 이동의 편리함(개방) vs 액체 보존(밀폐)
- **물리적 모순**: 컵은 열려 있어야 하고(마시기), 닫혀 있어야 한다(흘림 방지)

### IFR (이상적 최종 결과)
"커피컵이 스스로 흘림을 방지하면서, 마실 때는 쉽게 마실 수 있다"

### 발명원리 적용

| 원리 | 적용 | 아이디어 |
|------|------|---------|
| #15 역동성 | 열림/닫힘 상태 변화 가능 | 슬라이딩 뚜껑 |
| #7 포개기 | 컵 안에 구조물 | 내부 칸막이로 흔들림 감소 |
| #3 국소적 품질 | 마시는 부분만 열림 | 작은 음용구 |

### 물리적 모순 해결
- **시간 분리**: 마실 때만 열림 -> 플립 캡
- **공간 분리**: 위만 열림 -> 작은 음용구가 있는 뚜껑
- **조건 분리**: 기울일 때만 열림 -> 압력 조절 뚜껑

### 최종 해결책
**트래블 머그 (Travel Mug)**: 슬라이딩 음용구 + 밀폐 뚜껑

## 8. 주의사항

### Do's
- 문제를 모순으로 재정의
- IFR(이상적 결과) 먼저 상상
- 여러 발명원리 조합 시도
- 시스템 내 자원 최대 활용
- 타협이 아닌 모순 제거 추구

### Don'ts
- 타협안(trade-off)에 안주
- 한 가지 원리만 고집
- 물리적 모순 무시
- 기존 지식/관행에 얽매이기
