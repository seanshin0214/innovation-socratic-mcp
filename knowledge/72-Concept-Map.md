## 1. 도구 개요

### 정의
개념도는 개념들과 그 관계를 시각적으로 표현하는 도구로, 노드(개념)와 링크(관계)로 구성된 네트워크 형태의 지식 표현 방법

### 핵심 원리
- **개념(Concepts)**: 사건이나 객체의 규칙성을 나타내는 단어
- **연결어(Linking Words)**: 개념 간 관계를 설명하는 동사/전치사
- **명제(Propositions)**: 개념 + 연결어 + 개념 = 의미 있는 진술
- **위계구조**: 상위 개념에서 하위 개념으로

### 언제 사용하나?
- 복잡한 주제 이해
- 지식 구조화
- 학습 및 암기
- 아이디어 정리
- 브레인스토밍 결과 조직화
- 팀 지식 공유

## 2. 개념도 구성요소

### 기본 구조
```
   ┌─────────┐
   │  개념 A  │
   └────┬────┘
        │ 연결어
        ▼
   ┌─────────┐
   │  개념 B  │
   └────┬────┘
        │ 연결어
        ▼
   ┌─────────┐
   │  개념 C  │
   └─────────┘
```

### 명제 예시
```
┌──────────┐          ┌──────────┐
│   물     │ ──────── │   산소   │
└──────────┘ 포함한다  └──────────┘

"물은 산소를 포함한다" = 명제
```

### 교차 연결 (Cross-Links)
다른 영역의 개념 간 연결
```
       개념 A ─────────────┐
         │                 │
         ▼                 │ 교차연결
       개념 B              ▼
         │              개념 X
         ▼                 │
       개념 C ◄────────────┘
```

## 3. 개념도 유형

### 3.1 위계형 (Hierarchical)
상위에서 하위로 전개
```
            동물
           /    \
        포유류   조류
        /  \      |
      개   고양이  독수리
```

### 3.2 거미형 (Spider)
중심에서 방사형 전개
```
           역사
            |
    문화 ── 문명 ── 기술
            |
           경제
```

### 3.3 순서형 (Flowchart)
순차적 프로세스
```
문제인식 → 정보수집 → 분석 → 결론
```

### 3.4 시스템형 (Systems)
순환 및 피드백 포함
```
입력 → 처리 → 출력
  ↑             │
  └── 피드백 ◄──┘
```

## 4. 진행 단계

### Step 1: 초점 질문 설정 (3분)
- 무엇에 대한 개념도인가?
- 예: "기후변화의 원인과 영향은 무엇인가?"

### Step 2: 개념 도출 (10분)
- 브레인스토밍으로 관련 개념 나열
- 포스트잇에 하나씩 작성
- 15-25개 개념 목표

### Step 3: 개념 위계화 (10분)
- 가장 포괄적인 개념 선택 (상단)
- 구체적인 개념으로 내려감
- 같은 수준 개념은 같은 높이

### Step 4: 연결 및 연결어 (15분)
- 개념 간 선으로 연결
- 각 연결에 연결어 작성
- 명제가 되는지 확인

### Step 5: 교차 연결 (10분)
- 다른 가지 간 연결 탐색
- 창의적 통찰 발견
- 새로운 관계 표시

### Step 6: 검토 및 수정 (10분)
- 명제 검증
- 누락된 개념/연결 추가
- 시각적 정리

## 5. 좋은 개념도 기준

| 요소 | 설명 | 점검 |
|------|------|------|
| 개념 | 단어/구로 명확하게 | ☐ |
| 연결어 | 동사/전치사로 관계 표현 | ☐ |
| 명제 | 읽으면 완전한 문장 | ☐ |
| 위계 | 상위→하위 논리적 | ☐ |
| 교차연결 | 영역 간 통합 | ☐ |

## 6. 연결어 예시

| 관계 유형 | 연결어 예시 |
|----------|-----------|
| 포함 | 포함한다, 구성된다 |
| 인과 | 야기한다, 결과한다 |
| 특성 | 이다, 특징이다 |
| 예시 | 예를 들면, 같은 |
| 기능 | 하는 역할, 사용된다 |
| 비교 | 비슷하다, 다르다 |
| 순서 | 먼저, 다음에 |

## 7. 실전 예시: 학습

### 초점 질문
"효과적인 학습의 요소는 무엇인가?"

### 개념도
```
              ┌──────────────────┐
              │    효과적 학습    │
              └────────┬─────────┘
                       │ 요구한다
         ┌─────────────┼─────────────┐
         ▼             ▼             ▼
   ┌──────────┐ ┌──────────┐ ┌──────────┐
   │   동기   │ │   전략   │ │   환경   │
   └────┬─────┘ └────┬─────┘ └────┬─────┘
        │ 포함       │ 포함       │ 포함
        ▼            ▼            ▼
   ┌─────────┐ ┌─────────┐ ┌─────────┐
   │내적동기 │ │반복학습 │ │조용한곳 │
   └─────────┘ └─────────┘ └─────────┘
   ┌─────────┐ ┌─────────┐ ┌─────────┐
   │외적동기 │ │간격반복 │ │적절한  │
   └─────────┘ └─────────┘ │ 도구   │
        │           ▲      └─────────┘
        └───────────┘
         강화한다 (교차연결)
```

## 9. 개념도 vs 마인드맵

| 구분 | 개념도 | 마인드맵 |
|------|--------|----------|
| 구조 | 네트워크 | 방사형 |
| 연결어 | 필수 | 없음 |
| 교차연결 | 가능 | 드묾 |
| 목적 | 관계 이해 | 아이디어 확장 |
| 위계 | 논리적 | 자유로움 |

## 10. 주의사항

### Do's
- 연결어를 반드시 작성
- 명제로 읽어서 검증
- 교차연결 적극 탐색
- 여러 번 수정/발전

### Don'ts
- 연결어 없이 선만 긋기
- 너무 많은 개념 (30개 초과)
- 단순 목록처럼 나열
- 한 번에 완성하려 함
