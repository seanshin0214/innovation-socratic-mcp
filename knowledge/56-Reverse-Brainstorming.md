# Reverse Brainstorming (역브레인스토밍)

## 개요
"어떻게 문제를 해결할까?" 대신 "어떻게 문제를 더 악화시킬까?"를 질문하여, 그 반대 방향에서 해결책을 도출하는 창의적 사고 기법입니다.

## 핵심 원리

```
일반 브레인스토밍              역브레인스토밍
        │                           │
        ▼                           ▼
 "어떻게 성공할까?"          "어떻게 실패할까?"
        │                           │
        ▼                           ▼
   해결책 직접 도출      →     실패 방법 도출
                                    │
                                    ▼
                              각 항목을 뒤집어
                                    │
                                    ▼
                              → 해결책 도출
```

## 왜 효과적인가?

### 심리적 장벽 제거
- "좋은 아이디어"에 대한 부담 없음
- 비판적 사고 활용 (자연스러움)
- 재미있고 참여도 높음

### 새로운 관점
- 문제의 반대편 탐색
- 숨겨진 가정 발견
- 예상치 못한 해결책

## 진행 단계

### Step 1: 문제 정의
원래 문제를 명확히 정의

**예시**: "고객 만족도를 높이려면?"

### Step 2: 문제 뒤집기
문제를 반대로 재정의

**예시**: "고객을 최대한 화나게 하려면?"

### Step 3: 역방향 아이디어 생성
뒤집힌 문제에 대한 아이디어 브레인스토밍

**예시**:
- 전화를 안 받는다
- 문제를 인정하지 않는다
- 오래 기다리게 한다
- 불친절하게 대응한다
- 약속을 안 지킨다

### Step 4: 아이디어 반전
각 역방향 아이디어를 다시 뒤집기

**예시**:
| 역방향 아이디어 | → | 해결책 |
|---------------|---|--------|
| 전화를 안 받는다 | → | 3콜 내 응답 원칙 |
| 문제 인정 안 함 | → | 즉각적 사과와 인정 |
| 오래 기다리게 함 | → | 대기 시간 5분 이내 |
| 불친절하게 대응 | → | 친절 교육 강화 |
| 약속 안 지킴 | → | 약속 이행 추적 시스템 |

### Step 5: 해결책 발전
반전된 아이디어를 구체화하고 발전

## 혁신이론 적용

### 혁신 실패 방법 → 성공 전략

| 질문 | 역브레인스토밍 적용 |
|------|-------------------|
| 원래 질문 | "혁신을 성공시키려면?" |
| 뒤집힌 질문 | "혁신을 확실히 실패시키려면?" |

### 혁신 실패 레시피 예시

```
"혁신을 실패시키려면..."

1. 고객 의견을 무시한다 → 고객 중심 혁신
2. 모든 것을 한번에 바꾼다 → 점진적 실험
3. 실패를 처벌한다 → 실패 허용 문화
4. 사일로를 유지한다 → 부서 간 협업
5. 단기 성과만 본다 → 장기 비전 수립
6. 리더가 관심 없다 → 리더십 몰입
7. 자원을 안 준다 → 충분한 투자
8. 변화를 강요한다 → 참여와 동기부여
```

## 활용 예시

### 예시 1: 직원 이직률 문제

**원래**: "직원 유지율을 높이려면?"
**뒤집기**: "직원들이 최대한 빨리 퇴사하게 하려면?"

| 퇴사 유도 방법 | → | 유지 전략 |
|--------------|---|----------|
| 성과 인정 안 함 | → | 즉각적 인정과 보상 |
| 성장 기회 없음 | → | 경력 개발 프로그램 |
| 과도한 업무량 | → | 워크-라이프 밸런스 |
| 소통 부재 | → | 정기 1:1 미팅 |

### 예시 2: 제품 품질 개선

**원래**: "제품 품질을 높이려면?"
**뒤집기**: "최악의 품질을 만들려면?"

| 품질 악화 방법 | → | 품질 향상 전략 |
|--------------|---|--------------|
| 검사 건너뛰기 | → | 다단계 품질 검사 |
| 싼 재료 사용 | → | 프리미엄 원자재 |
| 교육 안 함 | → | 지속적 기술 교육 |
| 피드백 무시 | → | 고객 피드백 시스템 |

## 워크샵 진행 팁

### 분위기 조성
- "바보 같은 아이디어 환영"
- 웃음과 유머 장려
- 비판 금지

### 시간 배분
1. 문제 정의 (3분)
2. 문제 뒤집기 (2분)
3. 역방향 브레인스토밍 (10분)
4. 아이디어 반전 (10분)
5. 발전 및 선정 (10분)

## 활용 상황
- 아이디어가 막힐 때
- 문제가 복잡할 때
- 팀 에너지가 필요할 때
- 새로운 관점이 필요할 때
- 위험 요소 파악할 때

## 장점
- 재미있고 참여적
- 심리적 장벽 낮음
- 비판적 사고 활용
- 숨겨진 문제 발견
- 창의적 해결책
