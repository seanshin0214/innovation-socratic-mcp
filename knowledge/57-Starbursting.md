# Starbursting (5W1H 질문 생성법)

## 개요
아이디어나 문제에 대해 Who, What, When, Where, Why, How 질문을 체계적으로 생성하여 모든 측면을 탐색하는 브레인스토밍 기법입니다.

## 핵심 원리

```
                        What? (무엇)
                           │
                           │
           Why? (왜) ──────┼────── When? (언제)
                           │
                    ┌──────┴──────┐
                    │   아이디어   │
                    │   /문제     │
                    └──────┬──────┘
                           │
          Where? (어디서) ──┼────── Who? (누가)
                           │
                           │
                        How? (어떻게)
```

## 6가지 핵심 질문

### Who? (누구)
| 질문 유형 | 예시 질문 |
|----------|----------|
| 주체 | 누가 이것을 사용하는가? |
| 대상 | 누구에게 영향을 미치는가? |
| 이해관계자 | 누가 관련되어 있는가? |
| 책임자 | 누가 결정하는가? |
| 협력자 | 누구와 협력해야 하는가? |

### What? (무엇)
| 질문 유형 | 예시 질문 |
|----------|----------|
| 정의 | 이것은 정확히 무엇인가? |
| 구성 | 무엇으로 구성되는가? |
| 결과 | 무엇을 달성하는가? |
| 필요 | 무엇이 필요한가? |
| 장애 | 무엇이 문제인가? |

### When? (언제)
| 질문 유형 | 예시 질문 |
|----------|----------|
| 시작 | 언제 시작하는가? |
| 종료 | 언제 완료되는가? |
| 빈도 | 얼마나 자주 발생하는가? |
| 기한 | 데드라인은 언제인가? |
| 조건 | 어떤 상황에서 발생하는가? |

### Where? (어디서)
| 질문 유형 | 예시 질문 |
|----------|----------|
| 장소 | 어디서 일어나는가? |
| 시장 | 어떤 시장을 대상으로 하는가? |
| 채널 | 어디서 접근 가능한가? |
| 환경 | 어떤 환경에서 작동하는가? |
| 범위 | 어디까지 영향을 미치는가? |

### Why? (왜)
| 질문 유형 | 예시 질문 |
|----------|----------|
| 목적 | 왜 이것이 필요한가? |
| 동기 | 왜 사람들이 원하는가? |
| 원인 | 왜 이런 상황이 발생했는가? |
| 가치 | 왜 중요한가? |
| 정당성 | 왜 이 방법을 선택했는가? |

### How? (어떻게)
| 질문 유형 | 예시 질문 |
|----------|----------|
| 방법 | 어떻게 작동하는가? |
| 프로세스 | 어떻게 실행하는가? |
| 측정 | 어떻게 성공을 측정하는가? |
| 개선 | 어떻게 더 좋게 만드는가? |
| 비용 | 얼마나 비용이 드는가? |

## 혁신이론 적용

### 혁신 프로젝트 Starbursting

**주제**: 새로운 혁신 프로젝트

| 질문 | 혁신 적용 |
|------|----------|
| Who? | 누가 혁신을 주도하는가? 누가 수혜자인가? |
| What? | 무엇을 혁신하는가? 무엇이 바뀌는가? |
| When? | 언제 시작/완료하는가? 적시성은? |
| Where? | 어디에 적용하는가? 어떤 시장인가? |
| Why? | 왜 이 혁신이 필요한가? 가치는? |
| How? | 어떻게 실행하는가? 어떻게 측정하는가? |

## 진행 방법

### Step 1: 중심 주제 정의
- 아이디어, 제품, 프로젝트, 문제 선정
- 별 중앙에 배치

### Step 2: 6개 가지 그리기
- 별 모양으로 6개 방향
- 각 가지에 W/H 질문 카테고리

### Step 3: 질문 생성
- 각 카테고리별 5-10개 질문 생성
- 모든 질문 환영, 판단 보류
- 시간 제한 (카테고리당 3-5분)

### Step 4: 질문 정리
- 중복 제거
- 우선순위화
- 핵심 질문 선정

### Step 5: 답변 및 액션
- 핵심 질문에 대한 답변
- 답변 불가 질문 = 조사 필요 영역
- 액션 아이템 도출

## 활용 예시

### 예시: 신제품 론칭

**주제**: 새로운 모바일 앱 론칭

| 카테고리 | 질문 |
|----------|------|
| Who? | 타겟 사용자는 누구인가? 경쟁자는? |
| What? | 핵심 기능은? 차별화 포인트는? |
| When? | 론칭 시기는? 개발 일정은? |
| Where? | 어떤 앱스토어에? 어느 국가부터? |
| Why? | 왜 사용자가 필요로 하는가? |
| How? | 어떻게 마케팅? 어떻게 수익화? |

## 워크샵 진행 팁

### 준비물
- 큰 종이 또는 화이트보드
- 6색 포스트잇
- 마커펜

### 시간 배분
1. 주제 정의 (3분)
2. 질문 생성 (20분, 카테고리당 3분)
3. 정리 및 우선순위 (10분)
4. 답변 토론 (15분)

## 활용 상황
- 신제품/서비스 기획
- 프로젝트 초기 단계
- 문제 분석
- 아이디어 검증
- 인터뷰 질문 준비
- 비즈니스 계획

## 장점
- 포괄적 탐색
- 빠짐없는 검토
- 구조화된 사고
- 팀 협업 용이
- 숨겨진 이슈 발견
