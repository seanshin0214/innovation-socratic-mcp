# 사고 도구 적용 가이드 (Thinking Tools Application Guide)

## 개요
상황과 목적에 맞는 적절한 사고 도구를 선택하기 위한 가이드입니다. 20가지 이상의 사고 도구를 상황별로 분류하고 활용 방법을 안내합니다.

## 상황별 추천 도구

### 1. 문제 원인 분석
**"왜 이런 문제가 발생했는가?"**

| 도구 | 용도 | 적합한 상황 |
|------|------|------------|
| **5 Whys** | 근본 원인 추적 | 단순한 문제, 빠른 분석 |
| **Fishbone Diagram** | 다차원 원인 분석 | 복잡한 문제, 팀 분석 |
| **Force Field Analysis** | 추진력/저항력 분석 | 변화 장애 요인 파악 |

### 2. 아이디어 발산
**"새로운 아이디어를 많이 만들어야 한다"**

| 도구 | 용도 | 적합한 상황 |
|------|------|------------|
| **SCAMPER** | 체계적 변형 | 기존 제품/서비스 개선 |
| **Brainwriting 6-3-5** | 침묵의 브레인스토밍 | 내성적 팀원 포함 |
| **Lotus Blossom** | 구조적 확장 | 아이디어 체계화 |
| **Reverse Brainstorming** | 역발상 | 아이디어 막힐 때 |
| **Synectics** | 유추/은유 | 파격적 발상 필요 |

### 3. 아이디어 수렴/정리
**"많은 아이디어를 정리하고 선택해야 한다"**

| 도구 | 용도 | 적합한 상황 |
|------|------|------------|
| **Affinity Diagram** | 그룹핑, 패턴 발견 | 정성 데이터 분류 |
| **PMI** | 장단점 평가 | 빠른 아이디어 검토 |
| **Six Thinking Hats** | 다관점 평가 | 심층 의사결정 |

### 4. 전략 수립
**"전략 방향을 설정해야 한다"**

| 도구 | 용도 | 적합한 상황 |
|------|------|------------|
| **SWOT Analysis** | 내외부 분석 | 전략 기획 시작점 |
| **PESTLE Analysis** | 거시환경 스캔 | 시장 진출, 사업 검토 |
| **Business Model Canvas** | 비즈니스 모델 설계 | 신사업, 스타트업 |

### 5. 관점 전환
**"새로운 시각이 필요하다"**

| 도구 | 용도 | 적합한 상황 |
|------|------|------------|
| **Six Thinking Hats** | 다중 관점 사고 | 편향 방지, 균형 |
| **Reverse Brainstorming** | 역발상 | 막힌 상황 타개 |
| **Synectics** | 유추적 사고 | 창의적 도약 필요 |

### 6. 고객 이해
**"고객을 깊이 이해해야 한다"**

| 도구 | 용도 | 적합한 상황 |
|------|------|------------|
| **Design Thinking** | 인간 중심 혁신 | 제품/서비스 개발 |
| **Jobs-to-be-Done** | 고객 목적 파악 | 니즈 발굴 |
| **Empathy Map** | 고객 심리 이해 | 페르소나 개발 |

### 7. 그룹 토론/협업
**"다수가 함께 논의해야 한다"**

| 도구 | 용도 | 적합한 상황 |
|------|------|------------|
| **World Café** | 대규모 대화 | 100명+ 참여 |
| **Brainwriting 6-3-5** | 균등 참여 | 6-8명 그룹 |
| **Starbursting** | 질문 생성 | 초기 탐색 단계 |

### 8. 의사결정
**"선택하고 결정해야 한다"**

| 도구 | 용도 | 적합한 상황 |
|------|------|------------|
| **PMI** | 빠른 평가 | 단순한 결정 |
| **Six Thinking Hats** | 종합 평가 | 중요한 결정 |
| **Force Field Analysis** | 변화 결정 | 도입/변화 결정 |

## 도구 선택 플로우차트

```
시작
  │
  ▼
┌─────────────────────────────────────┐
│         무엇을 하려고 하는가?         │
└─────────────────────────────────────┘
  │
  ├─ 문제 원인 찾기 ────────► 5 Whys, Fishbone
  │
  ├─ 아이디어 생성 ────────► SCAMPER, Brainwriting
  │
  ├─ 아이디어 정리 ────────► Affinity, PMI
  │
  ├─ 전략 수립 ───────────► SWOT, PESTLE
  │
  ├─ 관점 전환 ───────────► Six Hats, Synectics
  │
  ├─ 고객 이해 ───────────► Design Thinking, JTBD
  │
  ├─ 그룹 토론 ───────────► World Café, Brainwriting
  │
  └─ 의사결정 ────────────► PMI, Six Hats
```

## 도구 조합 (Tool Chaining)

### 조합 1: 문제 해결 여정
```
Fishbone → 5 Whys → Brainwriting → PMI → 실행
(원인분석) (심화)   (해결책발상)  (선정)
```

### 조합 2: 혁신 프로젝트
```
PESTLE → SWOT → Design Thinking → SCAMPER → Affinity
(환경)   (전략)  (고객이해)       (아이디어)  (정리)
```

### 조합 3: 전략 기획
```
PESTLE → SWOT → Force Field → Six Hats → 실행계획
(외부)   (통합)  (변화분석)    (의사결정)
```

### 조합 4: 대규모 혁신 워크숍
```
World Café → Affinity → SCAMPER → PMI → 투표
(대화)      (정리)     (발전)    (평가)
```

## 3대 핵심 도구 요약

### 1. Six Thinking Hats (de Bono)
**다중 관점 사고**
- 🤍 White: 사실/데이터
- ❤️ Red: 감정/직관
- 🖤 Black: 비판/리스크
- 💛 Yellow: 긍정/기회
- 💚 Green: 창의/대안
- 💙 Blue: 프로세스/통제

### 2. SCAMPER
**체계적 아이디어 발상**
- **S**ubstitute (대체)
- **C**ombine (결합)
- **A**dapt (적용)
- **M**odify (변형)
- **P**ut to other uses (다른 용도)
- **E**liminate (제거)
- **R**everse (역발상)

### 3. Fishbone Diagram (Ishikawa)
**원인 분석**
- People (인력)
- Process (프로세스)
- Technology (기술)
- Culture (문화)
- Resources (자원)
- Environment (환경)
