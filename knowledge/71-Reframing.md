## 1. 도구 개요

### 정의
리프레이밍은 문제나 상황을 바라보는 관점(프레임)을 의도적으로 변경하여 새로운 해결책이나 통찰을 발견하는 사고 기법

### 핵심 원리
- **프레임 인식**: 현재 프레임의 한계 인식
- **관점 전환**: 다른 각도에서 문제 조망
- **재정의**: 문제 자체를 새롭게 정의
- **가능성 확장**: 새로운 해결 공간 발견

### 언제 사용하나?
- 문제가 막혀있을 때
- 기존 해결책이 효과 없을 때
- 새로운 관점이 필요할 때
- 갈등 상황 해소
- 창의적 돌파구 필요
- 고정관념 탈피

## 2. 리프레이밍 기법

### 2.1 문제 재정의

| 원래 문제 | 리프레이밍 |
|----------|-----------|
| "매출이 떨어진다" | "고객이 우리를 떠나는 이유는?" |
| "직원들이 게으르다" | "동기부여가 부족한 원인은?" |
| "시간이 없다" | "우선순위 정리가 필요하다" |
| "예산이 부족하다" | "자원을 더 효율적으로 쓸 방법은?" |

### 2.2 주체 전환

**질문**: 이 상황을 ___가 본다면?

| 주체 | 관점 |
|------|------|
| 고객 | 고객 입장에서 이 문제는? |
| 경쟁자 | 경쟁자가 이 문제를 어떻게 볼까? |
| 아이 | 5살 아이라면 뭐라고 할까? |
| 외계인 | 처음 보는 존재라면? |
| 미래 자아 | 10년 후 나라면? |

### 2.3 시간 전환

| 시간 프레임 | 질문 |
|------------|------|
| 과거 | 비슷한 상황을 어떻게 해결했나? |
| 현재 | 지금 당장 할 수 있는 것은? |
| 미래 | 5년 후에 이 문제가 중요할까? |
| 역사 | 100년 전에는 이것이 문제였을까? |

### 2.4 범위 전환

```
줌 아웃 (Zoom Out)
├── 이 문제가 더 큰 시스템에서 갖는 의미는?
├── 전체 맥락에서 보면?
└── 근본 원인은 무엇인가?

줌 인 (Zoom In)
├── 문제의 가장 작은 단위는?
├── 가장 먼저 해결할 것은?
└── 구체적으로 무엇이 문제인가?
```

### 2.5 긍정/부정 전환

| 원래 | 전환 |
|------|------|
| 문제 | 기회 |
| 약점 | 잠재력 |
| 실패 | 학습 |
| 제약 | 창의성 촉발 |
| 위기 | 전환점 |

## 3. 6가지 리프레이밍 질문

### Question 1: 반대
"정반대로 생각하면?"
- 문제가 아니라 기회라면?
- 줄이는 게 아니라 늘린다면?
- 막는 게 아니라 허용한다면?

### Question 2: 비유
"이것을 ___에 비유하면?"
- 이 조직이 동물이라면?
- 이 프로젝트가 여행이라면?
- 이 문제가 병이라면 처방은?

### Question 3: 극단
"극단적으로 생각하면?"
- 예산이 무한하다면?
- 시간이 하루뿐이라면?
- 완벽하게 해결된 상태는?

### Question 4: 다른 분야
"___분야에서는 이것을 어떻게?"
- 의료 분야에서는?
- 스포츠에서는?
- 자연에서는?

### Question 5: 역할 교환
"___입장이 되어보면?"
- 내가 고객이라면?
- 내가 신입이라면?
- 내가 CEO라면?

### Question 6: 시간 여행
"___년 후/전에서 보면?"
- 10년 후에 이걸 어떻게 볼까?
- 지금이 1900년이라면?
- 미래에서 온 나라면?

---

## 4. 진행 단계

### Step 1: 현재 프레임 명확화 (5분)
- 현재 문제를 어떻게 정의하고 있는가?
- 암묵적 가정은 무엇인가?
- 사용하는 언어/용어는?

### Step 2: 프레임 한계 인식 (5분)
- 이 프레임으로 보지 못하는 것은?
- 배제되는 해결책은?
- 왜 이 프레임에 갇혔는가?

### Step 3: 대안 프레임 생성 (15분)
- 2번 섹션의 기법들 적용
- 최소 3-5개 대안 프레임 생성
- 가장 다른 관점 탐색

### Step 4: 새 프레임 탐색 (10분)
- 각 프레임에서 보이는 것은?
- 새로운 해결 가능성은?
- 가장 유망한 프레임은?

### Step 5: 통합 (10분)
- 여러 프레임의 통찰 종합
- 새로운 문제 정의 도출
- 다음 행동 결정

---

## 5. 실전 예시

### 문제: "회의가 너무 많다"

**현재 프레임**: 회의 수를 줄여야 한다

**리프레이밍들**:

| 기법 | 새 프레임 | 통찰 |
|------|----------|------|
| 반대 | 회의를 더 효과적으로 | 질의 문제일 수도 |
| 주체 전환 | 참석자 입장 | 불필요 참석이 문제 |
| 범위 확대 | 소통 시스템 전체 | 비동기 소통 부재 |
| 긍정 전환 | 협업 기회 | 회의 목적 재정의 |
| 극단 | 회의 0개라면? | 필수 회의만 남김 |

**새로운 문제 정의**:
"어떻게 하면 회의 없이도 충분히 소통하고, 필수 회의는 짧고 효과적으로 할 수 있을까?"

## 6. 리프레이밍 프롬프트

### AI 활용 프롬프트
```
"다음 문제를 5가지 다른 관점에서 리프레이밍해주세요.

문제: [문제 설명]

각 리프레이밍에서:
1. 새로운 문제 정의
2. 이 관점에서 보이는 것
3. 가능한 해결 방향
을 제시해주세요."
```

## 7. 주의사항

### Do's
- 현재 프레임을 먼저 명확히
- 가장 다른 관점 시도
- 판단 없이 여러 프레임 탐색
- 불편한 프레임도 고려

### Don'ts
- 첫 번째 대안에서 멈추기
- 익숙한 프레임만 고수
- 리프레이밍 자체가 목적화
- 행동 없이 관점만 바꾸기
