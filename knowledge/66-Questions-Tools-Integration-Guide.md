# 질문과 사고 도구 통합 가이드

## 개요
1,040개 리더십 질문과 20+ 사고 도구를 연결하여 **트레이닝 모드**와 **실전 문제해결**에 활용하는 방법을 안내합니다.

---

## 1. 1,040개 질문 체계 구조

### 질문 분류

```
┌────────────────────────────────────────────────────────────┐
│                    1,040개 리더십 질문                       │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  PART 1: Descriptive Questions (250개)                     │
│  ├─ A. 인사/성과 관리 (30)                                  │
│  ├─ B. 조직 갈등/권력투쟁 (30)                               │
│  ├─ C. 윤리적 딜레마 (30)                                   │
│  ├─ D. 위기 관리/커뮤니케이션 (30)                           │
```

## 2. 질문-도구 매핑

### 상황별 도구 추천

| 질문 카테고리 | 추천 사고 도구 | 활용 방법 |
|--------------|---------------|----------|
| 인사/성과 관리 | 5 Whys, Force Field | 근본 원인 파악, 변화 저항 분석 |
| 조직 갈등 | Six Hats, World Café | 다관점 분석, 대화 촉진 |
| 윤리적 딜레마 | PMI, PERSPECTIVE | 장단점 평가, 다관점 윤리 분석 |
| 위기 관리 | Pre-Mortem, Fishbone | 사전 리스크 파악, 원인 분석 |
| 전략적 의사결정 | SWOT, PESTLE, Scenario | 환경 분석, 시나리오 기획 |
| 혁신/변화 | SCAMPER, Synectics | 창의적 아이디어 생성 |
| 문화/글로벌 | PERSPECTIVE, 세대별 관점 | 다문화 관점 이해 |

### 도구별 연결 질문

| 도구 | 트리거 질문 유형 |
|------|----------------|
| **5 Whys** | "왜 이런 문제가 발생했는가?" |
| **Fishbone** | "이 결과에 영향을 미친 요인들은?" |
| **Six Hats** | "다른 관점에서 보면?" |
| **SCAMPER** | "어떻게 개선/변형할 수 있는가?" |
| **Force Field** | "변화를 촉진/저해하는 요인은?" |
| **SWOT** | "우리의 강점/약점/기회/위협은?" |
| **Pre-Mortem** | "만약 실패한다면 왜?" |

## 3. 트레이닝 모드 활용법

### 3.1 개인 트레이닝

```
┌─────────────────────────────────────────────────────────┐
│                  개인 트레이닝 플로우                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 질문 선택 (1,040개 중 랜덤 또는 카테고리 선택)        │
│     │                                                   │
│     ▼                                                   │
│  2. 사고 도구 적용 (질문 유형에 맞는 도구 선택)            │
│     │                                                   │
│     ▼                                                   │
│  3. 답변 생성 (도구 프레임워크에 따라 체계적 답변)         │
│     │                                                   │
│     ▼                                                   │
│  4. 다관점 검토 (PERSPECTIVE 또는 Six Hats 적용)         │
│     │                                                   │
│     ▼                                                   │
│  5. 반성 및 학습 (어떤 통찰을 얻었는가?)                  │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 그룹 트레이닝

| 세션 | 활동 | 시간 | 도구 |
|------|------|------|------|
| 1 | 시나리오 질문 제시 | 5분 | 질문 DB |
| 2 | 개인 사고 | 10분 | 선택한 사고 도구 |
| 3 | 소그룹 토론 | 15분 | World Café / Brainwriting |
| 4 | 전체 공유 | 10분 | Affinity Diagram |
| 5 | 종합 및 학습 | 10분 | PMI / Six Hats Blue |

### 3.3 시나리오 기반 학습

**예시 시나리오**:
> "핵심 인재가 경쟁사 3배 연봉 제안을 받았을 때 어떻게 설득하는가?"

**도구 적용 순서**:
1. **5 Whys**: 왜 이 인재가 떠나려 하는가?
2. **Fishbone**: 이탈 원인 요소 분석
3. **Six Hats**: 다관점 해결책 탐색
4. **SCAMPER**: 보상/환경 개선 아이디어
5. **Force Field**: 잔류 촉진/저해 요인
6. **Pre-Mortem**: 설득 실패 시나리오 분석

## 4. 실전 문제해결 워크플로우

### 4.1 단계별 도구 적용

```
문제 인식
    │
    ├─► [Question Storming] 질문 폭풍 (40개)
    │
    ▼
문제 정의
    │
    ├─► [5 Whys] 근본 원인
    ├─► [Fishbone] 원인 요소
    │
    ▼
분석
    │
    ├─► [SWOT] 내외부 분석
    ├─► [PESTLE] 거시환경
    ├─► [Force Field] 변화 동인
    │
    ▼
아이디어 생성
    │
    ├─► [SCAMPER] 체계적 변형
    ├─► [Synectics] 유추적 사고
    ├─► [Brainwriting] 팀 아이디어
    │
    ▼
평가
    │
    ├─► [Six Hats] 다관점 평가
    ├─► [PMI] 장단점 분석
    ├─► [Pre-Mortem] 리스크 예측
    │
    ▼
결정
    │
    ├─► [Decision Matrix] 다기준 평가
    ├─► [Pros-Cons-Fixes] 최종 검토
    │
    ▼
실행 계획
```

### 4.2 빠른 의사결정 (15분 버전)

| 단계 | 도구 | 시간 |
|------|------|------|
| 1 | Question Storming (핵심 5개) | 3분 |
| 2 | Six Hats (빠른 순환) | 6분 |
| 3 | PMI | 3분 |
| 4 | 결정 | 3분 |

## 5. 질문 심화 기법

### 5.1 Why-How 레이더

```
           WHY (목적/원인)
               ↑
               │
    WHAT IF ←──┼──→ WHAT
  (가정/가능성)  │   (현상/사실)
               │
               ↓
           HOW (방법/프로세스)
```

### 5.2 질문 심화 프레임워크

| 수준 | 질문 유형 | 예시 |
|------|----------|------|
| Level 1 | What (현상) | "무엇이 일어났는가?" |
| Level 2 | How (과정) | "어떻게 이런 일이 발생했는가?" |
| Level 3 | Why (원인) | "왜 이런 상황이 되었는가?" |
| Level 4 | What if (가정) | "만약 ~라면 어떻게 될까?" |
| Level 5 | So what (함의) | "그래서 이것이 무엇을 의미하는가?" |
| Level 6 | Now what (행동) | "이제 무엇을 해야 하는가?" |

### 5.3 메타 질문

- "우리가 묻지 않은 더 중요한 질문은?"
- "이 질문의 전제가 틀렸다면?"
- "이 질문을 다르게 표현하면?"
- "반대 질문은 무엇인가?"

## 6. 산업별 질문 적용

### 제조업

| 상황 | 추천 도구 | 핵심 질문 |
|------|----------|----------|
| 품질 문제 | Fishbone, 5 Whys | "불량의 근본 원인은?" |
| 생산성 | Force Field, SCAMPER | "효율 향상 방안은?" |
| 안전 사고 | Pre-Mortem, Six Hats | "예방 가능한 리스크는?" |

### 서비스업

| 상황 | 추천 도구 | 핵심 질문 |
|------|----------|----------|
| 고객 불만 | Empathy Map, 5 Whys | "고객이 진정 원하는 것은?" |
| 서비스 혁신 | Design Thinking, SCAMPER | "차별화 포인트는?" |
| 팀 갈등 | Six Hats, World Café | "다양한 관점을 어떻게 통합?" |

### 테크/스타트업

| 상황 | 추천 도구 | 핵심 질문 |
|------|----------|----------|
| 피봇 결정 | SWOT, Scenario | "전환의 근거와 리스크는?" |
| 제품 개발 | Design Thinking, SCAMPER | "PMF 달성 방법은?" |
| 투자 유치 | PERSPECTIVE, Pre-Mortem | "투자자 관점에서 약점은?" |

## 7. AI와 함께하는 질문-도구 통합

### AI 활용 프롬프트 예시

```
[상황 입력]
"신제품 출시가 3개월 지연되었습니다."

[도구 지정]
"5 Whys와 Six Hats를 결합하여 분석해주세요."

[기대 출력]
1. 5 Whys 분석 (근본 원인 5단계)
2. Six Hats 관점별 평가
3. 통합 인사이트
4. 추천 액션
```

### AI 보조 워크플로우

```
사용자 질문 입력
       │
       ▼
   상황 분석 (AI)
       │
       ├─► 적합한 도구 추천
       │
       ▼
   도구 기반 분석 (AI 보조)
       │
       ▼
   다관점 검토 (PERSPECTIVE)
       │
       ▼
   통합 인사이트 생성
```
