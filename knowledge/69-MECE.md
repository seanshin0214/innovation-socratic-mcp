## 1. 도구 개요

### 정의
MECE는 문제나 정보를 구조화할 때 사용하는 원칙으로, 각 범주가 서로 겹치지 않으면서(ME) 전체를 빠짐없이 포괄(CE)하도록 분류하는 방법

### 핵심 원리
- **Mutually Exclusive (상호 배타적)**: 각 범주 간 중복 없음
- **Collectively Exhaustive (전체 포괄적)**: 모든 가능성 포함
- **논리적 구조화**: 복잡한 문제를 체계적으로 분해
- **누락 방지**: 빠진 부분 없이 완전한 분석

### 언제 사용하나?
- 문제 구조화가 필요할 때
- 전략 수립 및 분석
- 프레젠테이션 구조 설계
- 의사결정 옵션 정리
- 데이터 분류 및 정리
- 컨설팅 보고서 작성

## 2. MECE의 두 가지 조건

### 상호 배타적 (Mutually Exclusive)
```
좋은 예: 성별 분류
  ├── 남성
  └── 여성
  (중복 없음)

나쁜 예: 활동 분류
  ├── 스포츠
  └── 축구  ← 스포츠에 포함됨 (중복)
```

### 전체 포괄적 (Collectively Exhaustive)
```
좋은 예: 연령대 분류
  ├── 0-17세
  ├── 18-64세
  └── 65세 이상
  (모든 연령 포함)

나쁜 예: 연령대 분류
  ├── 10대
  ├── 20대
  └── 30대
  (40대 이상 누락)
```

## 3. MECE 분류 방법

### 3.1 이분법 (Dichotomy)
가장 완벽한 MECE 구조

| 분류 | 예시 |
|------|------|
| A / Not A | 고객 / 비고객 |
| 내부 / 외부 | 내부 요인 / 외부 요인 |
| 긍정 / 부정 | 찬성 / 반대 |
| 통제 가능 / 불가능 | 통제 가능 / 통제 불가능 |

### 3.2 프로세스 분류
시간/순서 기반

```
시간: 과거 - 현재 - 미래
단계: 계획 - 실행 - 평가
구매: 인지 - 고려 - 구매 - 사용 - 재구매
```

### 3.3 구성요소 분류
전체를 부분으로 분해

```
매출 = 고객수 x 객단가 x 구매빈도
비용 = 고정비 + 변동비
조직 = 사업부A + 사업부B + 지원부서
```

### 3.4 기존 프레임워크 활용

| 프레임워크 | 구성 |
|-----------|------|
| 3C | Customer, Company, Competitor |
| 4P | Product, Price, Place, Promotion |
| SWOT | Strengths, Weaknesses, Opportunities, Threats |
| 5 Forces | 기존경쟁, 신규진입, 대체재, 공급자, 구매자 |

## 4. 진행 단계

### Step 1: 분석 대상 정의 (3분)
- 무엇을 분류/구조화할 것인지 명확히

### Step 2: 분류 축 선택 (5분)
- 이분법? 프로세스? 구성요소? 기존 프레임워크?
- 목적에 맞는 분류 방식 선택

### Step 3: 1차 분류 작성 (10분)
- 상위 범주 도출
- ME 확인: 겹치는가?
- CE 확인: 빠진 것은?

### Step 4: 검증 및 조정 (5분)
- 각 범주에 사례 대입해보기
- 어디에도 속하지 않는 케이스 확인
- 여러 범주에 속하는 케이스 확인

### Step 5: 필요시 하위 분류 (5분)
- 상위 범주를 더 세분화
- 각 하위 분류도 MECE 원칙 적용

## 5. MECE 검증 체크리스트

| 점검 항목 | 확인 |
|----------|------|
| 각 범주가 서로 겹치지 않는가? | ☐ |
| 모든 가능성이 포함되어 있는가? | ☐ |
| 어디에도 속하지 않는 예외가 있는가? | ☐ |
| 두 범주 이상에 속하는 케이스가 있는가? | ☐ |
| 분류 기준이 일관적인가? | ☐ |

## 6. 실전 예시

### 문제: 매출 하락 원인 분석

**MECE 구조:**
```
매출 하락 원인
├── 고객 수 감소
│   ├── 신규 고객 유입 감소
│   └── 기존 고객 이탈 증가
├── 객단가 하락
│   ├── 가격 인하
│   └── 저가 제품 비중 증가
└── 구매 빈도 감소
    ├── 계절 요인
    └── 경쟁사 대안 이용
```

### 검증:
- ME: 각 원인이 중복되지 않음
- CE: 매출 = 고객수 x 객단가 x 구매빈도 (완전함)

## 7. 흔한 실수

### 비MECE 예시와 수정

**문제: 직원 분류**

나쁜 예 (비MECE):
- 정규직 / 파트타임 / 여성 직원 / 영업팀
- (중복: 여성 정규직 영업팀원은 어디에?)

좋은 예 (MECE):
```
고용형태 기준:
└── 정규직 / 계약직 / 파트타임

성별 기준:
└── 남성 / 여성

부서 기준:
└── 영업 / 마케팅 / 개발 / 지원
```

## 8. 주의사항

### Do's
- 분류 전에 목적을 명확히
- 분류 축을 하나로 통일
- 실제 케이스로 검증
- 단순한 구조 선호

### Don'ts
- 여러 분류 축 섞지 않기
- 기타 항목 남발 금지
- 너무 세분화하지 않기
- 형식만을 위한 MECE 금지
